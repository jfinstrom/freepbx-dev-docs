# FastAGI Handling Patterns

FastAGI routes AGI scripts through a local socket server instead of spawning a new PHP process on every call. The framework handles the dialplan rewrite in `ext_agi`; the server lifecycle itself lives in the **core** module.

## Dialplan Rewrite (`ext_agi`)

When generating dialplan, `ext_agi::output()` checks whether FastAGI is enabled:

```php
class ext_agi extends extension {
	private static $prependFastAGI = null;

	function output() {
		if (is_null(static::$prependFastAGI)) {
			static::$prependFastAGI =
				\FreePBX::Modules()->moduleHasMethod("core", "fastAGIStatus")
				&& \FreePBX::Core()->fastAGIStatus();
		}
		$data = static::$prependFastAGI && !preg_match('/^agi:\/\//', $this->data)
			? "agi://127.0.0.1/" . $this->data
			: $this->data;
		return "AGI(" . $data . ")";
	}
}
```

The rewrite logic is straightforward:

| Condition | Output |
|-----------|--------|
| FastAGI enabled + plain path | `AGI(agi://127.0.0.1/{path})` |
| Already `agi://` URL | `AGI({data})` unchanged |
| FastAGI disabled | `AGI({data})` standard fork |

## FastAGI Status Check

```php
\FreePBX::Modules()->moduleHasMethod("core", "fastAGIStatus")
\FreePBX::Core()->fastAGIStatus()  // returns bool — core module method
```

`fastAGIStatus()` is implemented in the **core** module. The framework only does the dialplan rewrite — it doesn't run the server.

## AGI Script Bootstrap (standard AGI)

AGI scripts in `{ASTAGIDIR}` bootstrap FreePBX externally:

```php
#!/usr/bin/php -q
<?php
$bootstrap_settings['freepbx_auth'] = false;
include_once '/etc/freepbx.conf';
// Use $astman, FreePBX::create(), etc.
```

See [External Bootstrap](external-bootstrap.md).

`phpagi.php` in `amp_conf/agi-bin/` provides the AGI class with a `$socket` property for FastAGI socket communication.

## FastAGI Server Management

The "Core FastAGI Server" is referenced in framework i18n strings and `Console/Job.class.php` messages. PM2/process management is handled by the **core** module.

`fwconsole start` can start services including UCP (Node.js); FastAGI server start/stop is core-managed.

## Module AGI Files

Modules place AGI scripts in:

```
{module}/agi-bin/{script}
```

These get symlinked/copied to `{ASTAGIDIR}` during `fwconsole reload` (see [Reload and Apply Config Flow](reload-apply-config.md)).

When FastAGI is active, dialplan references become `agi://127.0.0.1/{module-script-path}`.

## Things to watch for

- Framework only rewrites dialplan — it does not implement the FastAGI server.
- `Core::fastAGIStatus()` must exist and return true for the rewrite to activate.
- AGI paths already using the `agi://` scheme are not double-prefixed.
- AGI scripts need external bootstrap (`freepbx.conf`).
- FastAGI server lifecycle is documented in the core module, not framework.
