# Astman Manager Integration

FreePBX talks to Asterisk through the Asterisk Manager Interface (AMI) via the `AGI_AsteriskManager` class. At bootstrap, the framework connects and exposes the instance as the global `$astman` and as a BMO property. You'll touch this whenever you need to reload Asterisk, read or write AstDB, or run raw manager commands.

## Bootstrap Connection

```php
// bootstrap.php
$astman = new AGI_AsteriskManager($bootstrap_settings['astman_config'], $bootstrap_settings['astman_events']);
$astman->connect(...);
FreePBX::create()->astman = $astman;
```

Skip the connection entirely with `$bootstrap_settings['skip_astman'] = true` — useful for AGI scripts and cron jobs that don't need live AMI access.

Config values come from `$amp_conf`: `ASTMANAGERHOST`, `AMPMGRUSER`, `AMPMGRPASS`.

## Access Patterns

```php
global $astman;                    // Legacy
$astman = \FreePBX::create()->astman;  // BMO
\FreePBX::Astman()                 // Self_Helper (if available)
```

Pick one style and stick with it in your module. The BMO accessor is the modern path.

## Key `AGI_AsteriskManager` Methods

These are the methods you'll reach for most often:

| Method | Purpose |
|--------|---------|
| `connect()` | Establish AMI connection |
| `connected()` | Check connection status |
| `Reload()` | Asterisk reload (used in `fwconsole reload`) |
| `app_exists($app)` | Check Asterisk application exists |
| `database_show($family)` | Read AstDB family |
| `database_put($family, $key, $val)` | Write AstDB entry |
| `database_get($family, $key)` | Read AstDB entry |
| `database_del($family, $key)` | Delete AstDB entry |
| `database_deltree($family)` | Delete AstDB family |
| `LoadAstDB()` | Preload AstDB into cache |
| `getDBCache()` | Get cached AstDB data |
| `command($cmd)` | Raw manager command |

There's also a `$useCaching` property that controls whether AstDB reads go through the in-memory cache.

## AstDB Helper

```php
astdb_get($exclude = array())  // utility.functions.php — read AstDB with optional family exclusion
```

AstDB is Asterisk's built-in key-value store. Modules use it for persistent dialplan data. During reload, FreePBX preloads it:

```php
$astman->useCaching = true;
$astman->LoadAstDB();
```

## Caching Pattern

If you're debugging AstDB reads, toggle caching to compare cached vs. uncached results:

```php
$astman->useCaching = false;
$uncached = $astman->database_show();

$astman->useCaching = true;
$cached = $astman->database_show();
// Should match when caching works correctly
```

## PKCS (SSL Certificate Management)

`FreePBX\PKCS` (`BMO/PKCS.class.php`) handles certificate generation for modules that need TLS:

| Method | Purpose |
|--------|---------|
| `setKeysLocation($loc)` | Set cert storage path |
| `createConfig($base, $cn, $o, $force)` | Create OpenSSL config |
| `generateKey($name, $password, $bits)` | Generate private key |
| `createCA($base, $passphrase, $force)` | Create CA cert |
| `createCSR($name, $params, $regen)` | Certificate signing request |
| `selfSignCert($name, $caname, $password, $serial)` | Self-sign certificate |

Access via `FreePBX::PKCS()`.

## Realtime (AstDB/Extconfig)

`FreePBX\Realtime` (`BMO/Realtime.class.php`) manages extconfig entries:

| Method | Purpose |
|--------|---------|
| `enableQueueLog()` | Add queue_log to extconfig.conf |
| `disableQueueLog()` | Remove queue_log from extconfig |
| `write()` | Write extconfig entries |

## Reload Integration

`fwconsole reload` requires a working AMI connection — there's no way around it:

```php
if (!$this->freepbx->astman->connected()) {
	throw new \Exception(_("Unable to connect to Asterisk Manager"));
}
// ...
$this->freepbx->astman->Reload();
```

## Things to watch for

- AMI connection is required for reload and real-time operations — scripts that skip it can't trigger Asterisk changes.
- The default manager password triggers a security notification (see [Reload and Apply Config Flow](reload-apply-config.md)).
- `$useCaching` affects AstDB read performance — it's enabled during reload for a reason.
- Use `skip_astman` in bootstrap settings for scripts that don't need AMI.
- PKCS operations require OpenSSL on the system.
- Astman is a global singleton — don't create duplicate connections.
